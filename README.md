# Загрузчик аудио с MTS Link (Webinar.ru)

Скрипт для скачивания и сведения аудиодорожек из записей звонков и вебинаров на платформе MTS Link (ранее Webinar.ru).

**Cервис МТС Линк чудовищен💩 Если вас заставляют его использовать на работе, вы знаете, какие паршивые транскрипты он делает, и что экспорт записей в файл просто не работает. Этот скрипт написан для тех, кому просто нужно выдрать аудио и загрузить его в [Whisper](https://github.com/ggml-org/whisper.cpp)**

Так как платформа МТС Линк хранит записи не в виде единого медиафайла, а как набор отдельных пересекающихся аудио- и видеопотоков (голоса разных спикеров, демонстрация экрана и т.д.), этот скрипт автоматизирует процесс их сборки. Он обращается к API платформы, получает таймлайн (хронологию) вебинара, скачивает все аудиодорожки и  склеивает их в единый аудиофайл с точным соблюдением задержек старта каждого спикера.

На выходе вы получаете качественный моно WAV-файл (16-bit, 16kHz), который идеально подходит для прослушивания или автоматической транскрипции нейросетью.

## Требования

Для работы скрипта необходимо наличие:
* **Python 3** (скрипт использует только стандартные библиотеки, поэтому `pip install` не требуется).
* **FFmpeg** (утилита для работы с аудио и видео).

### Установка FFmpeg:
* **macOS:** `brew install ffmpeg`
* **Ubuntu/Debian:** `sudo apt install ffmpeg`
* **Windows:** скачайте с [официального сайта](https://ffmpeg.org/download.html) (или используйте пакетный менеджер, например `winget install ffmpeg`).

## Использование

1. Откройте приложение MTS Link. Найдите и откройте запись прошедшей встречи. Нажмите "Поделиться". Выберите "доступ: для всех, у кого есть ссылка". Скопируйте полученную ссылку вида `https://my.mts-link.ru/12345/567890/record-new/11223344/record-file/987654321`
2. Скачайте файл `download_webinar.py` из этого репозитория.
3. Сделайте его исполняемым (для Linux/macOS):
   ```bash
   chmod +x download_webinar.py
   ```
4. Запустите скрипт, передав ему ссылку на запись вебинара:
   ```bash
   ./download_webinar.py "https://my.mts-link.ru/12345/567890/record-new/11223344/record-file/987654321"
   ```

Скрипт автоматически проанализирует ссылку, скачает необходимые потоки и создаст файл вида:
`webinar_{session_id}_{file_id}_{ДатаВремя}.wav` (например, `webinar_123_456_20260304_223015.wav`).

## Как это работает под капотом
1. Скрипт парсит предоставленный URL и извлекает `eventSessionId` и `recordFileId`.
2. Делает запрос к публичному API (`/api/event-sessions/.../flow`) для получения JSON с таймлайном.
3. Находит все события `mediasession.add`, содержащие пути к плейлистам `.m3u8` и их `relativeTime` (время старта потока относительно начала вебинара).
4. Загружает каждый аудиопоток локально в формате WAV.
5. Строит сложный фильтр `-filter_complex` для `ffmpeg` с использованием фильтра `adelay`, чтобы расставить дорожки по времени, а затем смешивает их фильтром `amix`.
6. Очищает временные файлы загруженных потоков.
